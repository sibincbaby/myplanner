# EchoCode

> Audio notifications for Claude Code: hear a spoken summary when a task finishes so you can walk away and stay informed

**Inspired by:** [claude-code-voice](https://github.com/jeanduplessis/claude-code-voice)  
**Date discovered:** 2026-07-22

---

## What gap it fills

During long Claude Code runs — a big refactor, a multi-file migration, an automated cron task — there's no feedback when the task finishes. You either stay glued to the terminal or miss the moment it's done. EchoCode hooks into Claude Code's `stop` event, reads the final assistant message, and speaks a 1–2 sentence summary aloud using either the macOS system voice or a local TTS model. You hear "Refactored 14 files. Three tests are failing in auth module." and walk back to the keyboard informed.

## Stack recommendation

| Layer | Choice | Why |
|-------|--------|-----|
| Hook trigger | Claude Code `stop` hook | Fires with transcript path after every session |
| Summary extraction | Python: parse last assistant message from JSONL | Simple; no API call needed |
| TTS (quick) | macOS `say` command | Zero dependencies; works out of the box |
| TTS (quality) | Kokoro TTS (local, offline) | 100 MB model, natural voice, no cloud |
| Optional LLM summary | Claude API with `haiku-4-5-20251001` | Cheap summarisation for long sessions |

## MVP scope (1 Claude session)

Three steps that compose into a working hook:
1. Claude Code `stop` hook reads the last N messages from the session transcript
2. Extracts the final assistant message (or calls Claude Haiku for a 1-sentence summary)
3. Calls `say` (macOS) or `pyttsx3` to speak it

Total code: ~80 lines of Python. Install: copy to `~/.claude/hooks/` and add to `settings.json`.

Out of scope for MVP: streaming announcements mid-session, custom voices, quiet-hours scheduling.

## Phases

### Phase 1 — Hook scaffold (0.5 h)
- Python script: `echocode.py` that reads `$CLAUDE_TRANSCRIPT_PATH` from env
- Parse JSONL and extract the last `role=assistant` content block
- Truncate to 200 characters; print to stdout (test without audio first)

### Phase 2 — System TTS (0.5 h)
- macOS: `subprocess.run(["say", "-r", "175", summary])`
- Linux: `subprocess.run(["espeak", summary])` fallback
- Windows: `pyttsx3` fallback

### Phase 3 — LLM summary (1 h)
- If final message is > 500 chars, call Claude Haiku: "Summarize this in one sentence for a spoken announcement: {text}"
- Config flag `USE_LLM_SUMMARY=true` in `~/.echocode/config.toml`
- Estimated cost per summary: ~$0.0002

### Phase 4 — Kokoro TTS (local quality voice) (1 h)
- Install Kokoro: `pip install kokoro-onnx sounddevice`
- Config option `TTS_BACKEND=kokoro` in config
- Falls back to `say` if Kokoro model not installed

### Phase 5 — Quiet hours + tone variants (0.5 h)
- `~/.echocode/config.toml`: `quiet_hours = ["22:00", "08:00"]`
- During quiet hours: play a short chime instead of speech
- Config: `success_chime` / `error_chime` paths for custom audio files

## Effort estimate

~2 hours for Phase 1–2 (MVP) · 1 Claude session  
~3.5 hours complete

## Blockers / risks

- **macOS only for quality TTS**: Kokoro requires ONNX runtime; `say` is macOS-specific. Mitigation: `espeak` + `pyttsx3` cover Linux/Windows as fallback.
- **Hook environment**: `$CLAUDE_TRANSCRIPT_PATH` must be set by Claude Code. Mitigation: test with Claude Code 2.x which exposes this env var.
- **Summary quality**: the last assistant message may be code, not prose. Mitigation: detect code blocks and substitute "Code output: N lines" when prose is missing.
