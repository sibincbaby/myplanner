# Claudish — Claude Code with Any of 580+ Models

**Inspired by:** [github.com/MadAppGang/claudish](https://github.com/MadAppGang/claudish)  
**Source:** GitHub Search / Claude Code model wrapper (v7.11.0 released June 30 2026, 928 stars)  
**Interest score:** 4/4 (Claude/LLM tooling + dev productivity + daily coding + novel multi-provider proxy)

## What It Is

A local TypeScript proxy server that sits between Claude Code and any LLM provider. You run `claudish` instead of `claude`, and it translates Anthropic API calls to OpenRouter, Gemini, OpenAI, Ollama, and 580+ other models — without changing Claude Code itself. Vision-incapable models get automatic image-to-description conversion. Provider is selected with a `@` prefix: `--model google@gemini-2.0-flash`.

## Why Build Your Own

Claudish is open-source, so you can build a stripped-down version you fully understand and control. Rolling your own means: you pick exactly which providers you care about (Gemini + Ollama is probably 80% of the use case), you add cost tracking per session, you add a model routing policy (e.g. "use Ollama for code tasks, Gemini for writing"), and you skip the 184-release upgrade churn. Buildable in one evening.

## Stack Recommendation

- **TypeScript + Bun** — fast startup, built-in bundler, same as the reference project
- **Hono** — lightweight web framework for the proxy HTTP server
- **Vercel AI SDK** (`@ai-sdk/google`, `@ai-sdk/openai`, `@ai-sdk/anthropic`) — unified provider interface
- **zod** — validate and transform request/response schemas

## MVP Scope

A local Hono server on `localhost:3456` that accepts Anthropic `/v1/messages` calls, re-routes them to Gemini or Ollama using Vercel AI SDK, and streams the response back in Anthropic SSE format. Wire it to Claude Code via `ANTHROPIC_BASE_URL=http://localhost:3456`.

## Implementation Phases

### Phase 1 — Proxy Server Skeleton
**Goal:** A Hono server that accepts Anthropic Messages API calls and echoes them back.

**Files:**
- `src/index.ts` — Bun entry point, starts Hono app
- `src/routes/messages.ts` — `POST /v1/messages` handler
- `src/types.ts` — Anthropic `MessagesRequest` type

**Key steps:**
1. `bun init claudish && bun add hono @hono/node-server zod`
2. Define `MessagesRequest` type matching Anthropic's schema (model, messages, max_tokens, stream, system)
3. `POST /v1/messages` parses body with Zod, logs the model name, returns a mocked streaming response
4. Mock response format: `data: {"type":"content_block_delta","delta":{"text":"hello"}}\n\ndata: [DONE]\n\n`
5. Start with `bun run src/index.ts`, set `ANTHROPIC_BASE_URL=http://localhost:3456 ANTHROPIC_API_KEY=fake` and run `claude --print "say hi"`

**Verify:** Claude Code sends request to localhost proxy, proxy logs it, Claude Code sees "hello" response.

---

### Phase 2 — Gemini Provider
**Goal:** Route `google@...` model names to Google Gemini via AI SDK.

**Files:**
- `src/providers/gemini.ts` — wraps `@ai-sdk/google` and streams responses
- `src/router.ts` — parses `model` field to extract `provider@model` syntax
- Updated `src/routes/messages.ts` — dispatch to provider

**Key steps:**
1. `bun add @ai-sdk/google ai`
2. `ModelRouter.parse("google@gemini-2.0-flash")` → `{provider: "google", model: "gemini-2.0-flash"}`
3. `GeminiProvider.stream(request)` converts Anthropic `messages[]` format → AI SDK `CoreMessage[]`, calls `streamText()`, yields Anthropic-format SSE chunks
4. Map Anthropic roles: `user`→`user`, `assistant`→`assistant`, `system` stays as system prompt
5. Stream `content_block_delta` events and finish with `message_stop`

**Verify:** `--model google@gemini-2.0-flash claude --print "what is 2+2"` returns `4` via Gemini.

---

### Phase 3 — Ollama Provider
**Goal:** Route `ollama@...` model names to a local Ollama instance.

**Files:**
- `src/providers/ollama.ts` — direct HTTP calls to Ollama's OpenAI-compatible endpoint

**Key steps:**
1. Ollama exposes `POST http://localhost:11434/v1/chat/completions` (OpenAI-compatible)
2. `OllamaProvider.stream(request)` converts Anthropic format → OpenAI format, streams from Ollama, converts back to Anthropic SSE
3. Add `OLLAMA_BASE_URL` env var (default `http://localhost:11434`)
4. Handle Ollama's non-streaming `done: true` termination vs Anthropic's `[DONE]`

**Verify:** With Ollama running `llama3.2`, run `--model ollama@llama3.2 claude --print "hello"` → response from local model.

---

### Phase 4 — Vision Fallback (Image → Description)
**Goal:** When a vision-incapable model receives an image in the message, convert it to a text description first using a vision model.

**Files:**
- `src/middleware/vision-fallback.ts` — detects image content blocks, replaces with Claude vision description

**Key steps:**
1. Non-vision models: `VISION_FALLBACK_MODEL` env var (e.g. `google@gemini-2.0-flash` which supports vision)
2. Before routing, scan `messages[].content[]` for `{type: "image_url"}` blocks
3. If found and target model lacks vision: call `GeminiProvider` with just the image and prompt "Describe this image in detail for use as context", replace image block with `{type: "text", text: description}`
4. Log the substitution so users can see it happened

**Verify:** Take a screenshot, paste it into Claude Code chat using a non-vision Ollama model — proxy swaps it with Gemini's description and Ollama responds correctly.

---

### Phase 5 — Cost Tracking Dashboard
**Goal:** Track token usage and estimated cost per session, provider, and model.

**Files:**
- `src/db/usage.ts` — SQLite table for request logs
- `src/routes/dashboard.ts` — `GET /dashboard` returns HTML usage summary

**Key steps:**
1. `bun add better-sqlite3` and create `usage (id, ts, provider, model, input_tokens, output_tokens, est_cost_usd)`
2. After each proxied request, insert row with token counts from response `usage` field
3. Add pricing table for each provider/model (update monthly)
4. `GET /dashboard` renders a simple HTML table grouped by date → model with total cost

**Verify:** After 10 Claude Code sessions, visit `http://localhost:3456/dashboard` — see breakdown like "Gemini Flash: $0.003, Ollama: $0.00".

## Estimated Effort

**1 Claude session.** Phases 1–3 take ~2.5 hours and give a fully working multi-provider proxy. Phase 4 adds 45 min. Phase 5 is 1 hour.

## Potential Blockers

- **Anthropic SSE format changes**: Claude Code may break if the proxy doesn't emit `content_block_start` / `content_block_stop` events properly — test each tool-use flow
- **Model name collisions**: providers name models differently. Maintain a `MODEL_ALIASES` map (e.g. `"gemini-flash"` → `"gemini-2.0-flash-001"`)
- **Tool use / function calling**: Anthropic's tool format differs from OpenAI's. The proxy must convert `tool_use` blocks in both directions — skip for MVP, add in Phase 2+
- **Streaming backpressure**: Hono streams need `TransformStream` to avoid buffering the entire response before forwarding
