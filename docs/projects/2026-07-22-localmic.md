# LocalMic

> Local voice agent for Mac: hold a hotkey, speak, and either insert transcribed text into any field or hand off to a local Claude-connected agent — no cloud, no account

**Inspired by:** [Mispher](https://www.producthunt.com/products/mispher-agentic-transcription-for-mac) (102 upvotes, July 10 2026)  
**Date discovered:** 2026-07-22

---

## What gap it fills

Mispher is a polished closed-source Mac app costing $7/month. Its core idea is compelling: hold a hotkey anywhere on your Mac, speak, and pick what happens — transcribe into the focused field, rewrite the selected text, translate, or invoke an LLM agent. An open-source Python version gives you the same loop, total control over the local model, and a hook into your own MCP servers for the agent mode. No subscription, no telemetry.

## Stack recommendation

| Layer | Choice | Why |
|-------|--------|-----|
| Hotkey | `pynput` (cross-platform) | Captures global modifier keys; simple |
| Audio recording | `sounddevice` + `numpy` | Minimal, low-latency recording while key held |
| STT | `whisper.cpp` Python bindings or `faster-whisper` | Offline; small model (39 MB) fast enough for real-time |
| Text insertion | macOS Accessibility API via `pyobjc` | Inserts into any focused text field system-wide |
| Selected text | `subprocess("pbpaste")` + `pbcopy` | Capture selection for rewrite mode |
| Agent (optional) | Claude API via `anthropic` SDK + MCP client | Triggered only in "Ask" mode |

## MVP scope (1–2 Claude sessions)

Three modes, triggered by how long you hold the hotkey:
1. **Transcribe** (default): hold ⌥, speak, release → insert transcription at cursor
2. **Rewrite** (hold ⌥ + ⇧): highlight text, hold hotkey, speak "make this shorter" → replace selection with rewritten text
3. **Ask** (hold ⌥ for 1.5 s): speak a question → Claude API returns answer → insert or show in a popup

Out of scope for MVP: MCP server connection, translation, radial UI, tray icon.

## Phases

### Phase 1 — Hotkey + record (1.5 h)
- `pynput` listener: on `⌥` press start recording, on release stop
- `sounddevice.rec()` to a temp WAV file
- Print transcribed text to stdout (no insertion yet)

### Phase 2 — Local Whisper transcription (1 h)
- `faster-whisper` with `base.en` model (~150 MB)
- Transcribe temp WAV → string
- Target: < 1 second latency for 5-second utterances

### Phase 3 — Text insertion (1.5 h)
- macOS: `Quartz.CGEventCreateKeyboardEvent` to type the transcription character by character — or the simpler `subprocess("osascript -e 'tell application "System Events" to keystroke ...'")` approach
- Test across Terminal, VS Code, browser text fields

### Phase 4 — Rewrite mode (1 h)
- On hotkey press: `subprocess.run(["pbpaste"])` to capture selected text
- Whisper transcribes the spoken instruction
- Call Claude API: "Rewrite the following text according to this instruction: [{instruction}]\n\n{selected_text}"
- `pbcopy` the result + keystroke `⌘V` to replace selection

### Phase 5 — Ask mode + tray icon (1.5 h)
- Detect long press (> 1.5 s) as "Ask" mode
- Call Claude API, show response in a floating `tkinter` or `rumps` popup
- `rumps` menubar app for start/stop/mode toggle

## Effort estimate

~4 hours for Phases 1–3 (MVP) · 1 Claude session  
~6.5 hours complete with rewrite and ask modes

## Blockers / risks

- **macOS accessibility permissions**: text insertion via Accessibility API requires granting the Terminal/app explicit permissions in System Settings → Privacy → Accessibility. Mitigation: document in README; graceful error if permission missing.
- **Whisper latency**: on older Macs without Apple Silicon, `base.en` may take 2–3 s. Mitigation: use `tiny.en` model (39 MB) for sub-500 ms latency; trade accuracy.
- **Key conflict**: `⌥` is used in many apps for special characters. Mitigation: make the trigger hotkey configurable in `~/.localmic/config.toml`.
- **pynput on macOS 15+**: may require accessibility permission for key event monitoring. Mitigation: detect and prompt user at first run.
