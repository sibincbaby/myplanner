# VoxDAG

> Local voice-to-structured-log agent for daily capture — speak thoughts, tasks, or ideas; Claude transcribes and structures them into a timestamped daily entry with extracted tasks, mood, and links; everything stored locally

**Inspired by:** [xiaotianfotos/homerail](https://github.com/xiaotianfotos/homerail) (GitHub trending July 24 2026, 664 stars)  
**Date discovered:** 2026-07-24

---

## What gap it fills

Homerail proves the voice-first local agent pattern: speak → structured output → auditable replay. But Homerail targets homelab DAG orchestration; the simplest daily-utility version of the same idea is a voice diary that turns a 30-second voice note into a structured daily log entry. No app to open, no typing — just press a hotkey, speak, and get back a Markdown entry with `## Summary`, `## Tasks`, `## Mood`, and `## Links` sections, appended to today's `YYYY-MM-DD.md` in your journal folder. Optional: a Flutter mobile companion for capture on the go.

## Stack recommendation

| Layer | Choice | Why |
|-------|--------|-----|
| Voice capture (desktop) | `sounddevice` + WAV buffer or `pyaudio` | Cross-platform mic access, no dependencies |
| Transcription | Whisper.cpp (local, `faster-whisper` Python wrapper) or Groq Whisper API | Local-first; Groq is instant fallback if CPU is slow |
| Structuring | Claude API (claude-haiku-4-5 for speed/cost) | Parses transcript into structured Markdown sections |
| Storage | `~/journal/YYYY-MM-DD.md` append-only | Plain Markdown, Obsidian/Logseq compatible |
| Hotkey trigger | `pynput` global hotkey listener or `rumps` (macOS menu bar) | One keypress to start recording |
| Mobile (optional) | Flutter + `record` package + Supabase Edge Function | iOS/Android voice capture syncing to same journal |

## MVP scope (1 Claude session)

A Python CLI script `voxdag` that:

1. On invocation: records from mic until silence is detected (or user presses Enter)
2. Transcribes with `faster-whisper` (small model, local)
3. Calls Claude to structure the transcript into a Markdown entry
4. Appends the structured entry to `~/journal/$(date +%Y-%m-%d).md`
5. Prints the result to stdout

No daemon, no hotkey, no UI for MVP — just `voxdag` in the terminal.

## Phases

### Phase 1 — Voice capture (45 min)
- Record from default mic using `sounddevice` into a WAV buffer
- Silence detection: stop recording after 1.5 s of silence below threshold (or manual Enter)
- Save to a temp WAV file
- Optional: play a soft "ding" on start/stop so you know it's listening

### Phase 2 — Transcription (30 min)
- `faster-whisper` with `model_size="small"` (runs on CPU in ~2-4 s for 30 s audio)
- Falls back to Groq Whisper API if `--fast` flag is passed
- Returns plain text transcript
- Log transcript to stdout with `[transcript]` prefix for debugging

### Phase 3 — Claude structuring (45 min)
- Prompt: "Structure this voice note into a journal entry with ## Summary (1-2 sentences), ## Tasks (bullet list of action items, or empty), ## Mood (1-word emoji + label), ## Links (any URLs or names mentioned). Keep it concise. Transcript: {text}"
- Use `claude-haiku-4-5` for speed (< 1 s, < 0.01 USD per entry)
- Return structured Markdown string

### Phase 4 — Journal append (30 min)
- Target file: `~/.voxdag/journal/YYYY-MM-DD.md` (configurable via `VOXDAG_JOURNAL_DIR`)
- Append with a `---` separator and `> 🎙️ HH:MM` timestamp header
- Create the file + add YAML front-matter date if it doesn't exist yet
- Print confirmation: "Appended to ~/journal/2026-07-24.md (4 tasks extracted)"

### Phase 5 — Global hotkey + tray icon (1.5 h)
- macOS: `rumps` menu-bar app — click mic icon to record, shows "Recording…" / "Processing…" states
- Linux: `pynput` global hotkey (`Ctrl+Alt+V`) triggers recording in background
- System notification (via `plyer`) on completion with entry preview
- Optional: `voxdag web` command launches a local readonly view of the journal (same stack as AgentSession)

## Effort estimate

~2.5 hours for Phases 1–4 (CLI MVP) · 1 Claude session  
~4.5 hours with hotkey + tray icon

## Blockers / risks

- **Whisper model download on first run**: `small` model is ~480 MB. Mitigation: print a "downloading model (480 MB, once only)…" message; cache in `~/.voxdag/models/`.
- **Mic permission on macOS**: Python scripts need explicit mic permission in System Preferences. Mitigation: wrap in a `try/except` and show a clear "Grant mic access to Terminal in System Prefs → Privacy" message.
- **Silence detection threshold varies by room noise**: fixed threshold misses pauses in noisy environments. Mitigation: calibrate first 0.5 s of recording as ambient baseline; use relative threshold (30% above baseline = silence boundary).
