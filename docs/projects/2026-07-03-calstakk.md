# CalStakk — Local-First Calendar & To-Dos Claude Can Actually Manage

**Inspired by:** [github.com/jpincas/calstakk](https://github.com/jpincas/calstakk)  
**Source:** GitHub Search / Claude skill (TypeScript + Deno, 1 star, active reconstruction)  
**Interest score:** 3/4 (Claude/LLM tooling + dev productivity + daily utility)

## What It Is

CalStakk is a self-hosted CalDAV + VTODO server with a web UI and, crucially, an **MCP layer (29 tools)** so an AI agent can read and mutate your real calendar and task list. It stays interoperable with Apple Reminders, Tasks.org, and Thunderbird via standard CalDAV — so the data isn't locked in a chat silo. The novel bit for this toolkit: a personal calendar/task store that Claude drives natively, instead of yet another chatbot with an ephemeral to-do list.

## Why Build Your Own

Full RFC 4791/5545 CalDAV compliance is overkill for a personal MVP. What you actually want is a small local task+event store with a tight MCP server: "what's on today", "add a task due Friday", "move my 3pm to 4pm", "what's overdue". Skip the CalDAV protocol server initially and expose a simple ICS export for read-only sync into your phone calendar. Owning it means your schedule lives on your machine and Claude can plan your day against it.

## Stack Recommendation

- **Deno + Hono** (or Node + Fastify) — single-file server, TS end-to-end
- **SQLite** — events + todos with recurrence fields
- **`@modelcontextprotocol/sdk`** — the MCP server Claude Code connects to
- **`ics` npm package** — generate an ICS feed for read-only phone sync
- **Minimal HTML + Tailwind** — a day/week view and a quick-add box
- Add CalDAV write-sync only later if you want two-way phone editing

## MVP Scope

A local server storing events and todos in SQLite, plus an MCP server registered in Claude Code exposing ~8 tools: `list_today`, `list_range`, `add_event`, `add_todo`, `complete_todo`, `reschedule`, `list_overdue`, `search`. A minimal web day-view. Claude can then plan and edit your day conversationally, and you subscribe your phone calendar to the ICS feed for glanceable read-only sync.

## Implementation Phases

### Phase 1 — Data Model + Store
**Goal:** SQLite store for events and todos with basic recurrence.

**Files:**
- `db/schema.sql` — `events`, `todos` tables
- `src/store.ts` — typed CRUD helpers

**Schema sketch:**
```sql
CREATE TABLE events (
  id TEXT PRIMARY KEY, title TEXT NOT NULL,
  start TEXT NOT NULL, end TEXT,           -- ISO 8601
  rrule TEXT,                              -- optional RFC5545 recurrence
  location TEXT, notes TEXT
);
CREATE TABLE todos (
  id TEXT PRIMARY KEY, title TEXT NOT NULL,
  due TEXT, done INTEGER DEFAULT 0,
  priority INTEGER DEFAULT 0, notes TEXT
);
```

**Key steps:**
1. CRUD helpers with Zod validation on input
2. Store timestamps as ISO 8601 UTC; keep a fixed local TZ for display
3. Seed a few events/todos for testing

**Verify:** A script inserts an event + todo and reads them back correctly.

---

### Phase 2 — MCP Server
**Goal:** Claude Code can list and mutate your schedule.

**Files:**
- `src/mcp.ts` — MCP server exposing the tool set
- `mcp.json` snippet for Claude Code registration

**Key steps:**
1. Register tools: `list_today`, `list_range`, `add_event`, `add_todo`, `complete_todo`, `reschedule`, `list_overdue`, `search`
2. Each tool validates args with Zod and calls `store.ts`
3. Return compact JSON (ids + human strings) so Claude can chain edits
4. `claude mcp add calstakk -- deno run -A src/mcp.ts`

**Verify:** In Claude Code: "add a dentist appointment tomorrow at 10am" then "what's on tomorrow?" → the event round-trips.

---

### Phase 3 — Web Day/Week View
**Goal:** A glanceable UI to see and quick-add without Claude.

**Files:**
- `src/web.ts` — HTTP routes rendering HTML
- `public/app.js` — quick-add box + complete-todo toggles

**Key steps:**
1. `/` shows today: timed events in a column, todos (overdue highlighted) in a sidebar
2. Quick-add box parses "Lunch with Sam 1pm" and "Buy milk !today" client-side into event/todo calls
3. Week view (`/week`) with 7 columns
4. Mark todo done inline (POST → refresh)

**Verify:** Add via web, see it in Claude via `list_today`, and vice versa.

---

### Phase 4 — ICS Export for Phone Sync
**Goal:** Read-only calendar subscription on your phone.

**Files:**
- `src/ics.ts` — build an ICS feed from events

**Key steps:**
1. `GET /feed.ics?token=<secret>` returns a VCALENDAR of upcoming events
2. Expand simple `rrule` recurrences for the next ~90 days
3. Subscribe iOS/Android calendar to the URL (via your tunnel)
4. Cache the feed for a few minutes to avoid rebuild churn

**Verify:** Subscribe your phone calendar to the URL → events added via Claude appear on the phone within the refresh window.

---

### Phase 5 — Recurrence + Natural-Language Reschedule
**Goal:** Handle "every weekday" events and fuzzy reschedules.

**Files:**
- `src/recurrence.ts` — minimal RRULE expansion
- extend `src/mcp.ts` — `reschedule` accepts relative phrasing

**Key steps:**
1. Support `FREQ=DAILY/WEEKLY` with `BYDAY` and `UNTIL` (enough for personal use)
2. `reschedule(id, "push to next Tuesday")` — let Claude resolve the date and pass an ISO target
3. `list_overdue` surfaces stale todos so Claude can nudge you
4. Guard against editing a single occurrence vs the whole series (ask which)

**Verify:** Create a "standup every weekday 9:30" recurring event → it shows on the right days in web + ICS.

## Estimated Effort

**1 Claude session.** Phases 1–2 (store + MCP) are ~2 hours and already deliver the core "Claude manages my calendar" value. Web view, ICS, and recurrence add ~2 hours. Skipping full CalDAV is what keeps this to a weekend.

## Potential Blockers

- **Timezone/DST bugs**: store UTC, display in one fixed local TZ, and test an event spanning a DST change before trusting reschedules.
- **Recurrence is a rabbit hole**: full RFC 5545 is enormous. Explicitly support only DAILY/WEEKLY + BYDAY + UNTIL and reject the rest — don't chase EXDATE/BYSETPOS for a personal tool.
- **ICS feed auth**: the feed URL leaks your whole schedule if guessed. Use a long random token in the path and rotate it if shared; never expose `/feed.ics` unauthenticated.
- **Single vs series edits**: reschedule/delete on a recurring event is ambiguous. Make the MCP tool require an explicit `scope: this|future|all` so Claude can't silently rewrite the series.
- **Two-way phone sync is hard**: real CalDAV write-sync (Apple Reminders editing back) is a large effort — keep phone sync read-only (ICS) unless you genuinely need it.
