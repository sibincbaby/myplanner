## What It Is

A fully local, cross-platform Flutter diary app that analyses your mood and writing patterns using Claude ŌĆö no cloud sync required. Write entries, get AI reflections and mood trends, and search your past thoughts. Flutter + Rust backend (via `flutter_rust_bridge`) makes it fast and privacy-first.

## Why Build Your Own

Moodiary is AGPL-3.0 and great ŌĆö but building your own means you design the Claude prompts for your journaling style, you pick the schema, and you avoid any future licensing friction. An MVP with Claude integration is one solid session.

## Stack Recommendation

- **Flutter** (stable channel) ŌĆö cross-platform UI, runs on Android/iOS/desktop
- **Drift** (SQLite ORM for Flutter) ŌĆö local-first storage
- **Claude API** (via `http` package or `anthropic_sdk_dart` if available) ŌĆö mood analysis, reflection prompts
- **speech_to_text** Flutter package ŌĆö voice journaling
- **fl_chart** ŌĆö mood trend visualisation
- **flutter_quill** or plain `TextField` ŌĆö rich text entry editor

## MVP Scope

Create a text entry with date/mood tag ŌåÆ save locally ŌåÆ Claude analyses sentiment and suggests a reflection ŌåÆ view past entries in a scrollable timeline. Voice input and charts are Phase 3+.

## Implementation Phases

### Phase 1 ŌĆö App Skeleton + Drift Database
**Goal:** A working Flutter app that saves and lists diary entries.

**Files:**
- `lib/main.dart` ŌĆö app entry point, MaterialApp
- `lib/database/database.dart` ŌĆö Drift database with `Entry` table
- `lib/screens/home_screen.dart` ŌĆö scrollable list of recent entries
- `lib/screens/new_entry_screen.dart` ŌĆö text field + save button

**Key steps:**
1. `flutter create moodiary && cd moodiary`
2. Add deps: `drift`, `drift_flutter`, `path_provider` to `pubspec.yaml`
3. Define `Entry` table: `id`, `body`, `moodTag`, `createdAt`, `aiReflection`
4. `HomeScreen` queries all entries ordered by `createdAt DESC`
5. `NewEntryScreen` writes to Drift on tap of save FAB

**Verify:** Launch on Android emulator, write "Had a good day", save, see it in list.

---

### Phase 2 ŌĆö Claude Mood Analysis
**Goal:** After saving, Claude analyses the entry and returns a mood tag + 1-sentence reflection.

**Files:**
- `lib/services/claude_service.dart` ŌĆö HTTP client wrapping Claude Messages API
- `lib/models/mood_analysis.dart` ŌĆö data class for parsed response

**Key steps:**
1. Add `http`, `flutter_dotenv` to `pubspec.yaml`; store API key in `.env`
2. In `ClaudeService.analyse(String entryText)`:
   - POST to `https://api.anthropic.com/v1/messages`
   - System: "You are a journaling coach. Return JSON: {mood: string, reflection: string}"
   - User: the entry text
3. Parse response JSON, update `Entry.aiReflection` and `Entry.moodTag` in Drift
4. Show a bottom sheet with the reflection after save

**Verify:** Write "Stressed about the project deadline", save ŌĆö Claude returns `{mood: "anxious", reflection: "..."}`

---

### Phase 3 ŌĆö Voice Input
**Goal:** Speak a diary entry instead of typing.

**Files:**
- `lib/widgets/voice_recorder.dart` ŌĆö mic button widget
- Updated `new_entry_screen.dart`

**Key steps:**
1. Add `speech_to_text` to `pubspec.yaml`
2. Request microphone permission via `permission_handler`
3. Stream transcription into the text field in real time
4. Show waveform animation using `AnimatedContainer` while listening

**Verify:** Tap mic, speak a sentence, see it transcribed in the entry field.

---

### Phase 4 ŌĆö Mood Trends Chart
**Goal:** Visualise mood over the past 30 days as a coloured bar chart.

**Files:**
- `lib/screens/insights_screen.dart`
- `lib/widgets/mood_chart.dart`

**Key steps:**
1. Add `fl_chart` to `pubspec.yaml`
2. Query Drift for entries in last 30 days, group by `moodTag`
3. Map mood tags to colours (green=happy, yellow=neutral, red=stressed, blue=sad)
4. Render `BarChart` with day on x-axis, mood colour on y-axis

**Verify:** After 5+ entries with different moods, Insights tab shows a coloured chart.

---

### Phase 5 ŌĆö Search + Export
**Goal:** Full-text search across all entries; export to markdown.

**Files:**
- `lib/screens/search_screen.dart`
- `lib/services/export_service.dart`

**Key steps:**
1. Drift FTS extension or simple `LIKE %query%` for search
2. Highlight matching terms in results
3. Export all entries as a single `.md` file to device Downloads via `share_plus`

**Verify:** Search "deadline" returns the stressed entry; Export creates a readable markdown file.

## Potential Blockers

- **Claude API key on mobile**: never bundle in the APK. Use `flutter_dotenv` for dev, and proxy through a tiny backend (e.g. Cloudflare Worker) for production
- **Drift codegen**: run `dart run build_runner build` after changing the schema ŌĆö easy to forget
- **iOS microphone permission**: add `NSMicrophoneUsageDescription` to `Info.plist` before testing on device
- **`fl_chart` version conflicts**: pin to a specific version to avoid null-safety breakage
