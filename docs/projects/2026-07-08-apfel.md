## Overview

Apfel wraps the `FoundationModels` framework — Apple's on-device 3B LLM shipped with macOS 26 Tahoe — three ways:

1. **CLI tool:** `apfel "what is the capital of France?"` reads from stdin/args and writes to stdout. Pipeable: `cat myfile.swift | apfel "explain this"`.
2. **OpenAI-compatible HTTP server:** `apfel serve` starts a local server at `127.0.0.1:11434/v1/chat/completions` — drop-in for any tool that accepts an OpenAI API base URL.
3. **Interactive chat:** `apfel chat` — persistent context window, tab completion, slash commands.

**Specs:** 3B parameter model, 4096-token context, 100% on-device (no network calls), no API key, no model download (ships with macOS). Requires macOS 26 Tahoe + Apple Silicon M1+.

**Why this matters for MCP:** any MCP client that accepts a custom OpenAI-compatible base URL (including Claude Desktop with a provider shim, Cursor, Continue, Open Interpreter) can be pointed at `http://localhost:11434` and will use the free on-device model. This makes it a free testing target for MCP server development and a zero-cost fallback for lightweight tasks.

## Stack Recommendation

For a personal build:

- **Language:** Swift 6.3 (same as upstream; FoundationModels is a Swift-only framework).
- **CLI layer:** `ArgumentParser` (Apple's Swift package) for `apfel`, `apfel serve`, `apfel chat` subcommands.
- **HTTP server:** `swift-nio` + `Hummingbird` (lightweight, async/await native) for the OpenAI-compatible endpoint.
- **Streaming:** FoundationModels returns an `AsyncStream<String>` of tokens; map to SSE `data: {"choices":[{"delta":{"content":"..."}}]}\n\n` for `/v1/chat/completions` with `stream: true`.
- **Build output:** `swift build -c release` produces a single binary; `brew install --build-from-source` or a simple makefile copies it to `/usr/local/bin`.

## MVP Scope

**In scope:**
- `apfel "<prompt>"` — single-turn CLI (reads prompt from arg or stdin, prints response to stdout, exits).
- `apfel serve [--port N]` — local OpenAI-compatible server with `/v1/chat/completions` (streaming and non-streaming).
- `apfel chat` — interactive REPL with context accumulation until the 4096-token limit.
- Basic error handling: clear message if FoundationModels is unavailable (non-Tahoe, non-Apple-Silicon, Apple Intelligence disabled).

**Out of scope for MVP:** function calling / tool use, multi-modal (image input), persistent chat history, model selection (only one model is available anyway), the brew formula (use `make install` instead).

## Implementation Phases

### Phase 1: Single-turn CLI
**Goal:** `apfel "Hello"` prints the model's response and exits.

- Create `Package.swift`: `swift-tools-version: 6.0`, target `apfel`, dependency on `swift-argument-parser`.
- `Sources/apfel/main.swift`: `@main struct Apfel: AsyncParsableCommand` with `@Argument var prompt: String?`; if nil, read from `readLine()`.
- Import `FoundationModels`; `let session = LanguageModelSession()`; `let response = try await session.respond(to: prompt)`; `print(response.content)`.
- Build and test: `swift run apfel "what year is it?"`.

**FoundationModels API (Swift 6.3 / macOS 26):**
```swift
import FoundationModels

let session = LanguageModelSession()
let response = try await session.respond(to: "Hello, world")
print(response.content)

// Streaming:
for try await partial in session.streamResponse(to: prompt) {
    print(partial.content, terminator: "")
}
```

### Phase 2: OpenAI-compatible HTTP server
**Goal:** `apfel serve` starts a server; `curl http://localhost:11434/v1/chat/completions -d '{"model":"apple","messages":[{"role":"user","content":"hi"}],"stream":true}'` streams tokens.

- Add `Hummingbird` dependency to `Package.swift`.
- Parse OpenAI `ChatCompletionRequest` JSON (messages array → concatenate into a prompt string for now; later handle system/user/assistant roles properly).
- Non-streaming path: respond → format as `{"choices":[{"message":{"role":"assistant","content":"..."}}]}`.
- Streaming path: iterate `session.streamResponse()` → emit SSE chunks → final `data: [DONE]`.
- Add `GET /v1/models` returning `{"data":[{"id":"apple","object":"model"}]}` for client compatibility.

**Verify:** configure Cursor's local model as `http://localhost:11434` with model `apple`; run a simple completion.

### Phase 3: Interactive chat REPL
**Goal:** `apfel chat` starts a persistent session; context accumulates across turns.

- Create a `ChatSession` struct holding a `LanguageModelSession` with the `conversationalContext` option (if FoundationModels supports multi-turn natively) or a message history array manually concatenated into the prompt.
- REPL: `while let line = readLine()` → stream response → update context → repeat.
- `/clear` clears context; `/exit` quits; token count warning when approaching 4096.

### Phase 4 (optional): MCP server mode
**Goal:** `apfel mcp` exposes a `generate` tool via the MCP stdio transport, allowing Claude Code or any MCP client to call the on-device model as an MCP tool.

- Implement a minimal MCP stdio server (JSON-RPC over stdio) exposing one tool: `generate(prompt: string) → string`.
- Add to `claude_desktop_config.json` as a local MCP server.
- Use case: Claude Code asks the on-device model to pre-process or summarize a large file before passing the result upstream to Claude claude-sonnet-4-6.

## Potential Blockers

- **macOS Tahoe + Apple Silicon requirement:** this only runs on macOS 26 (Tahoe) with Apple Silicon (M1+) and Apple Intelligence enabled. Confirm the development machine qualifies before starting.
- **FoundationModels API changes:** the framework is new (2026); check the WWDC 2026 session notes and Swift package docs for the exact API surface — method names may differ from the training-data-inferred version above.
- **4096-token context window:** the on-device model's small context window makes it unsuitable for long documents. Mitigate with a pre-processing truncation step (summarize before sending) or by chunking input.
- **Concurrency in the HTTP server:** `LanguageModelSession` may not be safe to call concurrently. If it isn't, wrap requests in a serial `AsyncStream` or actor to serialize calls to the model.
- **Apple Intelligence must be enabled:** if the user has Apple Intelligence turned off in System Settings, `LanguageModelSession()` will throw. Surface a clear "Enable Apple Intelligence in System Settings → Apple Intelligence & Siri" error.
