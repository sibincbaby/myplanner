# Natively — Open-Source AI Meeting Assistant & Interview Copilot

**Inspired by:** [github.com/Natively-AI-assistant/natively-cluely-ai-assistant](https://github.com/Natively-AI-assistant/natively-cluely-ai-assistant)  
**Source:** GitHub Search / AI meeting assistant (v2.7.0 June 2026, 1.7k stars, 700+ DAU)  
**Interest score:** 4/4 (dev productivity + voice tools + diary/logging + novel free Cluely alternative)

## What It Is

A desktop Electron app that captures system audio and microphone in real time, transcribes via Whisper (<500ms latency), feeds context to Claude/Gemini/Ollama, and displays AI suggestions as an invisible overlay. Designed for meetings, technical interviews, and live note-taking. The original has 26 releases, stealth mode, local RAG over past meetings, screenshot OCR for coding problems, and 100% offline support via Ollama.

## Why Build Your Own

Building your own means: you skip the personal-use-only license and own the code outright, you tailor Claude's system prompt for your exact workflow (technical interviews vs. client meetings vs. lecture notes), you control exactly what gets stored locally and what gets sent to the cloud, and you can add integrations the reference project doesn't have (post to your Moodiary/diary, auto-create tasks in your MCP task manager). Two sessions to a working tool.

## Stack Recommendation

- **Electron + React + TypeScript** — cross-platform desktop shell
- **Tauri v2** — alternative if you prefer Rust-based smaller binary (Natively uses Electron; Tauri is lighter)
- **@anthropic-ai/sdk** — Claude Haiku for fast transcription summaries
- **whisper.cpp** (via Node.js bindings or a sidecar process) — local transcription
- **Node.js `speaker` + `mic` packages** — system audio capture (or Electron's `desktopCapturer`)
- **SQLite via better-sqlite3** — local meeting log + RAG storage

## MVP Scope

Electron app captures microphone audio → chunks into 5-second segments → transcribes via Whisper → sends rolling transcript to Claude Haiku → displays suggestion overlay. No stealth mode or OCR in MVP.

## Implementation Phases

### Phase 1 — Electron Shell + Audio Capture
**Goal:** A desktop window that captures microphone audio and shows a live waveform.

**Files:**
- `electron/main.ts` — BrowserWindow creation, IPC handlers
- `electron/audio-capture.ts` — Node.js `mic` package audio stream
- `renderer/src/App.tsx` — React waveform visualiser

**Key steps:**
1. `npx create-electron-app natively --template=typescript-webpack`
2. Add `mic` npm package for audio stream access
3. Open audio stream in main process, emit PCM chunks via `ipcMain` every 100ms
4. React renderer draws a simple `<canvas>` waveform from the PCM data
5. Add a Start/Stop toggle button

**Verify:** Launch app, click Start — see waveform animating while you speak into the microphone.

---

### Phase 2 — Whisper Transcription
**Goal:** Transcribe microphone audio to text in near-real-time.

**Files:**
- `electron/whisper-service.ts` — spawns `whisper.cpp` as a child process or calls `openai.audio.transcriptions`
- `renderer/src/components/Transcript.tsx` — scrollable live transcript panel

**Key steps:**
1. Easiest path: use OpenAI Whisper API (`openai.audio.transcriptions.create`) with 5-second WAV chunks
2. Accumulate PCM in a ring buffer; every 5 seconds flush to WAV (use `wav-encoder` npm package)
3. Send WAV buffer to Whisper, append text to transcript store
4. Show rolling transcript in a text area with auto-scroll

**Verify:** Speak "The quarterly revenue was three million dollars" — transcript shows it within 6 seconds.

---

### Phase 3 — Claude Suggestion Overlay
**Goal:** After each transcription chunk, Claude suggests a follow-up question, key point, or action item.

**Files:**
- `electron/claude-service.ts` — streaming Messages API call
- `renderer/src/components/Suggestion.tsx` — floating overlay panel

**Key steps:**
1. Add `@anthropic-ai/sdk` — use `claude-haiku-4-5-20251001` for lowest latency
2. Every 10 seconds (or on demand via hotkey), send last 60 seconds of transcript to Claude
3. System prompt: "You are a real-time meeting assistant. Based on this transcript snippet, provide: 1 key insight, 1 suggested follow-up question, and any action items. Be brief."
4. Stream the response into a transparent overlay window (Electron `BrowserWindow` with `transparent: true, frame: false, alwaysOnTop: true`)
5. Hotkey (e.g. `Ctrl+Shift+A`) to toggle overlay visibility

**Verify:** During a mock meeting, press hotkey — overlay appears with Claude's suggestion over your Zoom window.

---

### Phase 4 — SQLite Meeting Log + Local RAG
**Goal:** Every meeting is stored; Claude can search past meetings when answering questions.

**Files:**
- `electron/db.ts` — SQLite schema for meetings and transcript segments
- `electron/rag-service.ts` — FTS5 search over past meeting content

**Schema:**
```sql
CREATE TABLE meetings (id, title, started_at, ended_at, summary);
CREATE TABLE segments (id, meeting_id, ts, speaker, text);
CREATE VIRTUAL TABLE segments_fts USING fts5(text, content='segments');
```

**Key steps:**
1. On "End Meeting": generate a 3-sentence summary via Claude, save to `meetings` table
2. FTS5 virtual table over `segments` for keyword search
3. When Claude answers a question, prepend top-5 relevant past segments as context
4. Add "Past Meetings" sidebar showing recent meeting cards with search

**Verify:** After storing 3 meetings, ask "What did we decide about the API pricing?" — Claude finds the relevant meeting segment and cites it.

---

### Phase 5 — Export + Diary Integration
**Goal:** Export meeting notes as Markdown; optionally post summary to your Moodiary diary.

**Files:**
- `electron/export-service.ts` — Markdown/JSON export
- `electron/diary-integration.ts` — POST to Moodiary local HTTP API (if running)

**Key steps:**
1. "Export" button generates a Markdown file: title, date, participants (if manually added), summary, full transcript, action items
2. Use `shell.showSaveDialog` for native file picker
3. If Moodiary is running (check `localhost:3099/health`), offer "Save to Diary" which POSTs the meeting summary as a journal entry
4. Diary entry tagged `#meeting` with auto-linked action items as checkboxes

**Verify:** End meeting → click Export → `.md` file saved to Downloads → "Save to Diary" posts it to Moodiary.

## Estimated Effort

**2 Claude sessions.** Phase 1–2 (audio + transcription) takes ~2 hours. Phase 3 (overlay) takes 1 hour. Phases 4–5 each take 45–60 min.

## Potential Blockers

- **System audio capture on macOS**: `desktopCapturer` captures tab/window audio, not system-wide. Use BlackHole virtual audio device or the ScreenCaptureKit API via a native module.
- **Whisper API cost at 5-sec chunks**: at $0.006/min, 1-hour meeting = $0.36. For local, run `whisper.cpp` as a sidecar binary (faster than Python Whisper on Apple Silicon via Metal).
- **Electron transparency on Linux**: `transparent` BrowserWindow is compositor-dependent. Test on X11 vs. Wayland separately.
- **Latency budget**: Whisper chunk (5s) + API round-trip + Claude stream = ~8–10 seconds per suggestion. Use `claude-haiku-4-5-20251001` and stream Claude's response to start displaying within 1 second of receiving the first token.
