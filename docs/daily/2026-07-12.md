## Viable Projects

- **[Context Warp Drive — Deterministic, Cache-Preserving Context Folding for Claude Code](../projects/2026-07-12-context-warp-drive-deterministic-cache-preserving-context-folding.md)** — The only candidate to clear the bar (4/4). It compresses long agent transcripts by *deterministic structural folding* rather than LLM summarization, which keeps the provider prompt cache hot (claimed 92.6% cache-read hit rate, −60% cost vs truncation) — a genuinely different mechanism from the output-reduction skills the user already tracks (paleo, token-diet). The user burns tokens daily running discovery/agent workflows and actively manages context cost (ccplug, paleo), so a cache-preserving folder fills a real gap (fills_gap=1) they'd hit every session (daily_utility=1). The novelty is the non-LLM, cache-aware folding approach (novel=1), and the core loop — watch a Claude Code JSONL transcript, fold oldest turns at a token threshold while preserving the cached prefix, resume the session — is a focused, well-scoped weekend reimplementation in TypeScript (weekend_buildable=1). Upstream is early (62★, unpublished on npm), so it's a build-alongside rather than an install-and-forget.
