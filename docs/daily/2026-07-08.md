## Viable Projects

- **[nexu-io/open-design — open-source Claude Design alternative](../projects/2026-07-08-open-design.md)** — WB=1: the DESIGN.md-driven generator pattern is portable and a simplified personal fork with 10 design systems + Claude Code + Codex is achievable in 1-2 sessions. FG=1: no existing item in the toolkit covers AI-native prototype/landing-page generation from a design system schema. N=0: 40k+ stars and a polished v0.13.0 make this an established project rather than niche; the user would extend/adopt rather than clone from scratch. DU=1: any day involving a UI prototype, a landing page draft, or a slide deck would open this.

- **[kerlenton/mcpsnoop — "Wireshark for MCP"](../projects/2026-07-08-mcpsnoop.md)** — WB=1: a transparent stdio-piping proxy that logs JSON-RPC frames is exactly the kind of focused 200-line tool buildable in a single Claude session. FG=1: the official MCP Inspector connects as its own client; it never sees the actual messages a real agent (Claude Code, Codex, Cursor) sends the server. This in-path sniffer is the only way to observe production traffic. N=1: no other tool sits transparently in the real client↔server pipe. DU=1: every MCP server debugging session would open this first.

- **[Arthur-Ficial/apfel — free on-device AI on Mac](../projects/2026-07-08-apfel.md)** — WB=1: the Swift FoundationModels API is publicly documented and building a personal CLI wrapper around it (with an OpenAI-compatible shim) is a well-scoped weekend project. FG=1: free, zero-API-key, fully offline LLM available as a local server that any MCP client can point at is a missing primitive in the user's local tooling. N=1: using Apple Intelligence (FoundationModels framework, macOS Tahoe) as a CLI/server target is entirely new ground, not a clone of any existing OSS project. DU=1: would serve as a free always-available fallback model for offline/quota-exhausted sessions.

- **[jamiew/monarch-mcp — Monarch Money MCP for Claude](../projects/2026-07-08-monarch-mcp.md)** — WB=1: a FastMCP Python server wrapping a financial data API (Monarch Money or any YNAB/Plaid/CSV source) is a well-scoped 1-session build — the pattern is proven and the PyPI toolchain is fast. FG=1: the user's toolkit has general finance-AI entries (finance-agent, expense-budget-tracker, FinPilot) but nothing that gives Claude live read/write access to an actual connected financial account with multi-month trend analysis. N=1: the token-efficient compact transaction format, natural-language date parsing, and parallel multi-account fan-out distinguish this from the naive one-shot finance MCPs already catalogued. DU=1: a daily "what did I spend this week?" Claude query would open this every morning.
