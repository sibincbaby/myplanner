## Viable Projects

- **[radar-mcp — a personal discovery-feed MCP for Claude](../projects/2026-07-17-radar-mcp-personal-discovery-feed-mcp-for-claude.md)** — The most on-the-nose pick of the day: an MCP server that Claude Code queries to pull fresh AI/dev-project signals from public "new/trending" endpoints (GitHub search, HN Algolia, Reddit JSON), normalized and deduped against this repo's own `state/seen.json`. It scores 1/1/0/1 = 3. novel=0 is honest — TrendsMCP already exists and trend MCPs are common — but a personal, local MCP wired to the user's own seen-list and interest keywords fills a real gap in **this very toolkit** (today's discovery still runs on ad-hoc `WebSearch`) and is exercised every day the pipeline runs. Buildable in 1–2 sessions as a TypeScript stdio MCP over free, keyless APIs. weekend_buildable + fills_gap + daily_utility = 3, viable.

### Why the rest fell short

The user's toolkit is now dense: agent UIs (cloudcli, agents-ui, herdr), voice (voice-coder, hns), journaling (memex, moodiary), finance (finance-agent, expense-budget-tracker, monarch-mcp, frugon), and agent messaging (agent-messenger). That crowding is what caps most candidates at 2. **Stated** and **Amigo** are real finance apps but overlap the existing finance stack and analyze statements periodically, not daily (DU=0). **voice-to-claude** / **Talknotes** duplicate the voice kit (and voice-to-claude is macOS/Metal-only; the user is on Linux). **agent-os** / **Juggler** are agent UIs already covered by herdr + cloudcli, and are too large for a weekend. **agent-comms-mcp** and **AMA2** re-tread agent-messenger with heavier infra (Postgres/webhooks, a full messaging runtime).
