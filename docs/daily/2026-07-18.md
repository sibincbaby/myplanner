## Viable Projects

- **[retry-guard — a Claude Code hook that remembers failed fixes and blocks repeats](../projects/2026-07-18-retry-guard-claude-code-hook-that-blocks-repeating-failed-fixes.md)** — Seeded by *regressionledger*, the clearest on-profile pick of the day and the rare 4/4. Two small Claude Code hooks (`PostToolUse` to record, `PreToolUse` to guard) plus a per-project JSON ledger fingerprint each attempted fix, learn whether it actually passed, and block the agent before it re-applies a fix it already lost to. It is weekend_buildable (documented hook protocol, zero deps), fills a real gap in *this* toolkit (the user's own notes flag steps that silently no-op and get re-run — nothing they own remembers a failed fix), is genuinely novel (a failed-fix *memory* as a pre-edit guardrail — distinct from brain0's drift audit and statewright's state machine), and has daily utility (a passive always-on hook that helps every session it fires, at zero interaction cost). 1/1/1/1 = 4, viable.

### Why the rest fell short

The user's toolkit is now dense — agent UIs (herdr, cloudcli, agent-deck), voice (voice-coder, hns, openless), memory (memsync, afair, crystalline, live-memory), context savers (token-diet, paleo, context-warp-drive), finance (finance-agent, expense-budget-tracker, monarch-mcp, frugon) — and that crowding is what caps most of today's field at 1–2. **Syntra**, **klaatcode**, and **Hearth** are full multi-model control planes / coding agents / local assistants: too large for a weekend and already covered (herdr/omnigent, anyclaude/claudish, king-louie/sanna). **Ratel** and **Scribe** re-tread context-diet and agent-memory, both crowded and non-trivial to build well. **git-temp** and **Maxx** are genuinely small and daily-usable but fill no real gap — `.gitignore` already handles scratch files, and token counters (claudoro, frugon) already exist. **HeimWall** has a novel paste-time-secret-scan angle, but the user's flow is agent-driven (Claude reads files directly, rarely manual pastes), so it neither fills a gap nor gets daily use here.
