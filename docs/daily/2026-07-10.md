## Viable Projects

- **[puffinsoft/peek-cli — Claude Code sees the browser](../projects/2026-07-10-peek-cli.md)** — WB=1: a Chrome extension calling `captureVisibleTab()`, a localhost WebSocket broker, and a Claude Code skill — a single-evening bridge. FG=1: Playwright MCP screenshots its *own* headless browser; nothing in the toolkit lets the agent see the real, logged-in tab you have open. N=1: screenshot-only, from the live authenticated tab, delivered as a skill — that exact framing isn't catalogued. DU=1: every frontend session turns "does this look right?" into a one-line ask.

- **[Rodiun/frugon — retroactive LLM cost audit](../projects/2026-07-10-frugon.md)** — WB=1: a pass-through logging proxy plus a re-pricing analyzer over JSONL is a well-scoped 1–2 session build. FG=1: the toolkit has token reduction (token-diet) and live routing (claudish, anyclaude), but nothing that says "here's where you overpaid" from captured traffic. N=1: per-call counterfactual pricing with optional quality-sampling to confirm the cheaper model holds up is new ground. DU=0: an audit you run weekly/monthly, not daily — capture is continuous but analysis isn't.

- **[chrismbryant/claude-journal-mcp — a work journal that writes itself](../projects/2026-07-10-claude-journal-mcp.md)** — WB=1: markdown entries, ripgrep search, a session-stop hook — no ML, no DB, one session. FG=1: every memory tool catalogued (afair, crystalline, agentmemory, live-memory) serves the *agent's* recall; none is a human-facing daily work journal you read yourself. N=0: journal/memory plugins for Claude Code are a crowded lane — the zero-ML, hook-driven angle simplifies but doesn't create a new category. DU=1: a journal is inherently daily and auto-capture makes it accrue passively — "/journal search" answers "what did I do Tuesday?" at standup.
