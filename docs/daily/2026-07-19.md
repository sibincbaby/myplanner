## Viable Projects

- **[claude-branch — tree-based conversation branching for Claude Code](../projects/2026-07-19-claude-branch-tree-based-conversation-branching-for-claude-code.md)** — Seeded by *Juggler* (Show HN, 486★), a Go+Wails GUI coding agent where sessions form branching trees instead of linear transcripts. The build target is a lightweight local web UI that reads Claude Code's session files, renders the conversation as a branching tree, and lets you fork at any message — creating parallel exploration paths without losing the original thread. weekend_buildable=1 (reads existing session files, React tree renderer, single-file backend), fills_gap=1 (Claude Code sessions are linear — once you diverge you lose the thread — and no tool the user owns solves this), novel=1 (tree-based conversation navigation for coding agents is genuinely new and Juggler is still in early-alpha at v0.4.2), daily_utility=1 (every long Claude Code session involving exploration or iteration benefits from branching). 4/4.

- **[local-finance-mcp — local-first personal finance copilot as a Claude Code MCP](../projects/2026-07-19-local-finance-mcp-local-first-personal-finance-copilot.md)** — Seeded by *finance-assistant* (googlarz/taxde-skill, 33★), a Claude skill using real statutory tax math across 6 countries. The build target is a local-first MCP server: SQLite for transaction storage, CSV import for bank statements, and Claude as a natural-language interface to log expenses, query spending patterns, and run budget projections — all on-device, no cloud. weekend_buildable=1 (MCP server + SQLite + CSV parser is a well-charted 1-session build), fills_gap=1 (user has personal finance AI as a top interest and the existing finance tools in the toolkit — finance-agent, frugon, monarch-mcp — are either too broad or read-only dashboards, not conversational), novel=1 (local-first conversational finance with bank CSV import and no cloud is a genuinely open space — the 33★ project is the closest thing and it requires a specific skill install), daily_utility=1 (logging an expense by typing it into Claude is something you'd actually do daily). 4/4.

- **[managed-agent-bridge — MCP server to control Claude Managed Agents from any MCP client](../projects/2026-07-19-managed-agent-bridge-mcp-server-for-claude-managed-agents.md)** — Seeded by *managed-agent-control-mcp* (modus-agendi, 1★), a tiny community project that exposes the Claude Managed Agents beta API to MCP clients. The build target is a leaner, more focused version: an MCP server that lets you spawn Managed Agent sessions, stream their activity, and inject tool outputs — all from Claude Code itself, creating a meta-agent setup where Claude Code pilots Claude Managed Agents. weekend_buildable=1 (MCP server wrapping the Managed Agents REST API, documented protocol, same build pattern as the user's existing MCP skills), fills_gap=1 (Managed Agents is a new Anthropic feature and there is no first-party MCP bridge; the discovered project has 1★ and was just released), novel=1 (piloting Claude Managed Agents from inside Claude Code is a novel composition the ecosystem hasn't settled). Daily utility is 0 — Managed Agents aren't in every workflow — which is the one gap keeping this at 3/4.

### Why the rest fell short

**claude-sessions** is a 346-release, 2,688-commit beast built over months — replicating its walk-away queue, circuit breakers, Obsidian integration, and parallel worktrees is decidedly not a weekend project (WB=0), and the simpler subset the user could build lacks novelty since the project itself is already polished (N=0). **ai-cli-mcp** would duplicate the user's existing Claude Code setup — running Claude from within Claude via MCP adds indirection with no clear gain over direct invocation (FG=0, N=0). **Claude-Budgeting-Plugin** is a clean 7-star plugin but the finance toolkit is already crowded (finance-agent, expense-budget-tracker, monarch-mcp, frugon all landed in the past weeks) and the plugin's approach — a budgeting workspace provisioned by Claude — isn't novel enough in that context (N=0). **agent-tts** covers ground already occupied by AgentVibes (152★, 131 releases) and clarvis — the user has credible voice options already (FG=0, N=0). **mcp-ssh-manager** is a well-built DevOps tool but SSH orchestration is neither in the user's interest profile nor a daily-use need from this toolkit.
