# AI-Operator Idea Discovery — 2026-07-12

Shape of the bet: take an app that is genuinely powerful but that ordinary users bounce off of because the config/UI/learning-curve is too steep, and bolt on an AI natural-language "operator" — pre-loaded with skills to drive the app's real API/CLI, so the user just talks and the AI performs the actions.

Scouting note: self-hosted, SMB, and creative-video scouts returned clean; several other domain scouts hit provider rate limits, so a few cards below are synthesized from first-hand app knowledge. OPNsense resurfaced but is already in `idea-seen.json`, so it was dropped.

## 1. Keycloak — the identity server nobody can configure

- **Why powerful:** Full enterprise IAM — OIDC/SAML/OAuth2, realms, clients, roles, groups, identity brokering, user federation (LDAP/AD), and fully scriptable authentication flows. It's the open-source Auth0/Okta and runs a huge share of self-hosted SSO.
- **Why people bounce off:** The mental model is brutal — realms vs clients, client scopes, protocol mappers, composite roles, and the drag-and-drop authentication-flow builder. "Why is my token missing this claim / why does my redirect URI fail / how do I get groups into the JWT" is a permanent forum genre. Most people copy a tutorial and never understand the object graph.
- **AI operator:** An NL identity admin pre-loaded with the Keycloak Admin REST API and a model of the realm→client→mapper→role graph. It wires up an SSO integration, adds the right protocol mappers, configures federation, and — crucially — *diagnoses* a broken login by reading the live config and event log and explaining which piece is wrong.
- **Killer example:** user says *"Let my React app at app.acme.com log in with Google, and put the user's groups into the token"* → AI creates/updates the OIDC client with correct redirect URIs, adds the Google identity provider, creates a groups protocol mapper on a client scope, assigns it, and returns the client ID/secret and a test-login URL.
- **Integration path:** Keycloak Admin REST API (`/admin/realms/{realm}/clients`, `/identity-provider/instances`, `/client-scopes`, `/authentication/flows`), admin-cli token auth; also `kcadm.sh` and realm JSON import/export for idempotent config.
- **Scores:** demand **5**, buildability **4**.

## 2. Traefik — the reverse proxy that 404s for reasons no one can find

- **Why powerful:** Dynamic cloud-native reverse proxy / load balancer that auto-discovers services from Docker/Kubernetes/file providers, does automatic Let's Encrypt TLS, and offers routers, middlewares (auth, rate-limit, headers, redirects), and load balancing without a static config file.
- **Why people bounce off:** The label-based config is famously error-prone — the `traefik.http.routers.*` / `services.*` / `middlewares.*` namespace, entrypoints, cert resolvers, and the router↔service↔middleware wiring cause the classic "why is my container 404-ing / not getting HTTPS" spiral that fills the forums.
- **AI operator:** An NL operator pre-loaded with Traefik's dynamic-config schema and the Docker-label dialect. It generates correct labels or file-provider config, validates against the running router state via the Traefik API, and diagnoses why a service isn't routing or a cert didn't issue.
- **Killer example:** user says *"Expose my Vaultwarden container at vault.mydomain.com over HTTPS and only let me reach the admin page from my home IP"* → AI emits the exact Docker labels (router rule + host, TLS cert resolver, an IP-allowlist middleware on `/admin`), applies them, then reads `/api/http/routers` to confirm the router is healthy and the cert issued.
- **Integration path:** File/Docker dynamic-config providers (live YAML/TOML or container labels) plus the Traefik REST API (`/api/http/routers`, `/api/http/services`, `/api/http/middlewares`, `/api/overview`) for read-back and diagnosis.
- **Scores:** demand **5**, buildability **4**.

## 3. Cal.com — Calendly-on-steroids that hides its best features

- **Why powerful:** Self-hostable scheduling infrastructure — event types, availability schedules, round-robin/collective team events, buffers, booking limits, routing forms, reminder/follow-up workflows, webhooks, and app integrations.
- **Why people bounce off:** Advanced setups (round-robin team distribution, routing forms that qualify and route leads, multi-condition reminder workflows, per-event availability + buffers) live in deeply nested settings that non-technical owners mis-configure or never discover — so they use it as a dumb single-link calendar.
- **AI operator:** An NL scheduling operator pre-loaded with skills for event-type creation, availability rules, team round-robin config, routing-form logic, and reminder workflows. It assembles the full booking flow from a described business scenario.
- **Killer example:** user says *"Create a 30-minute paid intro call, only Tuesday/Thursday afternoons, round-robin across my two coaches, with an SMS reminder 1 hour before"* → AI creates the event type, availability schedule, team round-robin assignment, and the reminder workflow.
- **Integration path:** Cal.com REST API v2 (`/event-types`, `/schedules`, `/teams`, `/webhooks`, workflows endpoints) with API keys; self-host CLI/config for deployment.
- **Scores:** demand **5**, buildability **4**.

## 4. Adobe After Effects — motion power gated behind expressions

- **Why powerful:** Industry-standard motion graphics/compositing — animation, VFX, keying, tracking, and expression-driven procedural animation in one app. The object model exposes projects, comps, layers, keyframes, effects, expressions, and the render queue to scripting.
- **Why people bounce off:** Beginners drown in the interface and specifically bounce off pre-comps, the keyframe/graph editor, and expressions — the JavaScript expression language "looks intimidating if you've never written code," so most users copy-paste snippets they don't understand and give up on procedural animation.
- **AI operator:** An NL operator pre-loaded with ExtendScript skills for the AE object model — create/modify comps, add and time keyframes, apply/parameterize effects, wire expressions (wiggle, loops, parenting, time-remap), build lower-third/text rigs, drive the render queue. The user describes the motion; the AI generates and injects the `.jsx` that performs it, so they never touch the expression editor.
- **Killer example:** user says *"Make this logo bounce in from the top and settle, then gently wiggle forever"* → AI writes ExtendScript that adds position keyframes with overshoot easing and attaches a `wiggle()`-plus-decay expression to the logo layer, applied live in the open project.
- **Integration path:** ExtendScript (`.jsx`) against the AE scripting object model (`app.project`, `CompItem`, `Layer`, `Property`, expressions, `RenderQueue`) via the Scripts menu / ScriptUI panel or a CEP/UXP panel; well-documented at ae-scripting.docsforadobe.dev. (Same operator skill library ports to open-source node compositors — see Fusion/Natron mentions.)
- **Scores:** demand **5**, buildability **4**.

## 5. Invoice Ninja — powerful billing buried four menus deep

- **Why powerful:** Full self-hostable invoicing/billing platform — recurring invoices, quotes, expenses, projects, multi-gateway payments, client portals, and a complete REST API. Rivals paid SaaS like FreshBooks/Bill.com.
- **Why people bounce off:** SMB owners get lost in the sprawling settings (tax rates, payment terms, gateway config, recurring schedules, invoice-design templates) and the multi-step create-quote→convert→invoice→apply-payment flow. Forums are full of "how do I just do X" where X is buried four menus deep.
- **AI operator:** An NL bookkeeping clerk pre-loaded with the Invoice Ninja v5 API — create/send invoices, set up recurring billing, record payments, chase overdue clients, run AR aging, configure tax/terms. It maps plain requests to the entity graph (client → invoice → line items → payment) so the user never touches the settings tree.
- **Killer example:** user says *"Bill Acme Corp $2,400 for October consulting, net-15, and email it"* → AI finds/creates the Acme client, builds the invoice with the line item and net-15 due date, marks it sent, and triggers the email — one sentence instead of ~8 screens.
- **Integration path:** Invoice Ninja v5 REST API (`/api/v1/`, `X-API-TOKEN` auth) — `/invoices`, `/clients`, `/payments`, `/recurring_invoices`, `/quotes`, `/expenses`; plus webhooks.
- **Scores:** demand **4**, buildability **5**.

## 6. AdGuard Home — great blocker, unreadable query log

- **Why powerful:** Network-wide DNS server + ad/tracker blocker — filter lists, custom rules, per-client policies, DNS rewrites, upstream/DoH/DoT config, query logging, and safe-search/parental controls.
- **Why people bounce off:** Users install it fine but flounder on filter-rule syntax, per-client rules, DNS rewrites, and diagnosing *why* a legit site broke or an ad still shows. The query log is powerful but reading it to find the offending rule/list is a skill most lack.
- **AI operator:** An NL operator pre-loaded with the AdGuard Home REST API and filter-rule syntax. It adds/removes blocks by intent, creates per-device policies and DNS rewrites, and — the killer feature — reads the query log to explain and fix breakage ("this domain is blocked by list X; I've allowlisted it").
- **Killer example:** user says *"Instagram won't load on my phone since I set this up, and block YouTube on the kids' tablet after 9pm"* → AI searches the query log, finds the Instagram CDN domain blocked by a specific list and allowlists it; then creates a client entry for the tablet with a YouTube block and sets up the scheduled toggle.
- **Integration path:** AdGuard Home REST API (`/control/filtering/*`, `/control/rewrite/*`, `/control/clients/*`, `/control/querylog`, `/control/stats`) — clean JSON HTTP, plus editable YAML config.
- **Scores:** demand **4**, buildability **5**.

## 7. HashiCorp Vault — secrets management as a PhD

- **Why powerful:** The de-facto secrets manager — dynamic secrets, encryption-as-a-service, PKI/certificate issuance, database credential rotation, pluggable auth methods and secret engines, and fine-grained policy control.
- **Why people bounce off:** Nearly everything is a wall — the seal/unseal ritual, mounting and tuning secret engines, HCL policy syntax, auth-method roles, token TTLs/leases, and CLI-vs-API paths. Beginners store a static secret and never touch the actual power (dynamic creds, PKI) because the path there is opaque.
- **AI operator:** An NL security operator pre-loaded with Vault's API surface and a policy model. It enables/tunes the right engine, writes least-privilege HCL policies from a plain description, sets up an auth role, and stores/reads/rotates secrets — narrating what each grant allows.
- **Killer example:** user says *"Give my billing service read-only access to the Stripe key and nothing else"* → AI writes a scoped policy granting `read` on `secret/data/billing/stripe`, creates an AppRole bound to that policy, stores the key at that path, and returns the role-id/secret-id plus a one-line blast-radius explanation.
- **Integration path:** Vault HTTP API (`/v1/sys/mounts`, `/v1/sys/policies/acl`, `/v1/auth/*`, `/v1/secret/data/*`) with token/AppRole auth; mirrored by the `vault` CLI for every operation.
- **Scores:** demand **4**, buildability **4**.

## 8. Apache Superset — BI power gated behind SQL and a dense builder

- **Why powerful:** Enterprise BI — connect any SQL warehouse, build datasets, 40+ chart types, dashboards with cross-filters, row-level security, alerts, and a SQL Lab IDE. A real Tableau/Looker alternative.
- **Why people bounce off:** To get a chart you must register a database, define a dataset (often hand-writing SQL/virtual datasets), understand metrics vs columns vs calculated columns, then wrestle the chart builder's explore panel. Non-analysts stall at "I just want a bar chart of revenue by month."
- **AI operator:** An NL analyst pre-loaded with the Superset REST API plus schema introspection. It picks the right dataset (or creates a virtual one from generated SQL), builds the chart with correct metrics/dimensions/time-grain, and assembles a dashboard — all from a plain question.
- **Killer example:** user says *"Show me monthly revenue by product category for this year as a stacked bar, on a dashboard called Sales"* → AI introspects the warehouse, creates/reuses the dataset, POSTs a chart with the metric, category dimension, and month grain, then creates the "Sales" dashboard and places the chart.
- **Integration path:** Superset REST API (`/api/v1/database`, `/api/v1/dataset`, `/api/v1/chart`, `/api/v1/dashboard`) with JWT auth; SQL Lab endpoint for ad-hoc query generation.
- **Scores:** demand **4**, buildability **4**.

## 9. TrueNAS SCALE — ZFS power, permissions that don't stick

- **Why powerful:** ZFS-backed NAS/storage OS — datasets, snapshots, replication, SMB/NFS/iSCSI shares, encryption, scrub/SMART scheduling, and an app/container platform on top.
- **Why people bounce off:** ZFS concepts (datasets vs zvols, snapshot/replication tasks, ACL vs Unix permissions, dataset-level share config) overwhelm newcomers. People misconfigure permissions so shares aren't writable, or never set up snapshots/replication and lose data.
- **AI operator:** An NL storage operator pre-loaded with the TrueNAS middleware API and ZFS best-practice defaults. It provisions datasets with sane recordsize/compression, wires SMB shares with correct ACLs, and stands up snapshot + replication schedules for a real backup story.
- **Killer example:** user says *"Make a shared 'Family Photos' folder my wife and I can both write to over the network, and keep daily backups for a month"* → AI creates the dataset, sets a group + ACL granting both users write, creates and starts the SMB share, and configures a periodic snapshot task with 30-day retention.
- **Integration path:** TrueNAS JSON-RPC 2.0 over WebSocket (25.04+; `pool.dataset.*`, `sharing.smb.*`, `pool.snapshottask.*`, `replication.*`, `auth.login`); also driveable via the `midclt` client.
- **Scores:** demand **4**, buildability **4**.

## 10. Kong Gateway — API management by hand-editing entities

- **Why powerful:** High-performance API gateway — services, routes, consumers, and 60+ plugins (auth, rate-limiting, transformations, observability, CORS), declarative config, and multi-datacenter scale.
- **Why people bounce off:** The service→route→plugin→consumer model plus plugin config schemas is a lot; people mis-scope a plugin (global vs route), botch a JWT/key-auth setup, or can't reason about plugin execution order. "Why is my rate-limit not applying" is a support staple.
- **AI operator:** An NL API-platform operator pre-loaded with Kong's Admin API and plugin schemas. It registers upstreams, attaches routes, and configures the right plugin at the right scope from intent, then verifies against the live config.
- **Killer example:** user says *"Put my orders API behind an API key and cap each key at 100 requests a minute"* → AI creates the service+route, enables `key-auth` on that route, adds `rate-limiting` at 100/min, creates a consumer with a key, and returns the key + a curl test.
- **Integration path:** Kong Admin REST API (`/services`, `/routes`, `/plugins`, `/consumers`) or declarative `kong.yml` via decK; DB-less and DB modes both supported.
- **Scores:** demand **4**, buildability **4**.

## 11. Zammad — helpdesk automation locked behind rule builders

- **Why powerful:** Enterprise-grade helpdesk/ticketing — multi-channel email/chat/social, SLAs, triggers, scheduler jobs, macros, knowledge base, and reporting. Comparable to Zendesk, fully self-hosted and API-first.
- **Why people bounce off:** The power lives in "triggers," "scheduler jobs," "core workflows," and "macros" — object/condition/action rule builders with a steep mental model. SMB admins can't figure out how to automate routing, auto-replies, or escalations without deep doc diving.
- **AI operator:** An NL operator pre-loaded with skills to author triggers, macros, SLAs, and scheduler jobs. It maps a plain-English support policy into the correct condition/action config, creates it via API, and explains what it built.
- **Killer example:** user says *"When a VIP customer emails us and it mentions 'refund', tag it urgent, assign to the billing group, and send an auto-acknowledgement"* → AI creates the trigger with the right conditions/actions plus the macro/auto-reply text.
- **Integration path:** Zammad REST API (`/api/v1/triggers`, `/api/v1/macros`, `/api/v1/tickets`, `/api/v1/groups`, roles) with token auth.
- **Scores:** demand **4**, buildability **4**.

## 12. Logseq — outliner power locked behind query syntax

- **Why powerful:** Local-first outliner/PKM over plain Markdown — block references, backlinks, a Datalog-style query engine, task/PDF/whiteboard support, and a plugin API. A privacy-friendly Roam/Obsidian rival.
- **Why people bounce off:** The real power (linked queries, block refs, structured properties) requires learning `{{query ...}}` / Datalog-ish syntax and block-reference discipline. Most users end up with flat notes and never build the dynamic dashboards Logseq is famous for.
- **AI operator:** An NL knowledge operator that reads/writes the Markdown graph and generates the query blocks, page properties, and links behind the scenes — so the user asks for a view and gets a working live query, not a syntax lesson.
- **Killer example:** user says *"Make me a page that always shows every unfinished task tagged #work sorted by due date"* → AI creates the page and inserts a correct `{{query}}` (or advanced Datalog query) filtering `TODO/DOING` blocks with `#work` and a scheduled/deadline sort, and confirms it renders.
- **Integration path:** Plain Markdown files on disk (direct read/write) plus the Logseq plugin API / local HTTP API server for live block CRUD and query execution.
- **Scores:** demand **4**, buildability **4**.
