## 1. Vega-Lite — publication-grade charts trapped behind a JSON grammar

- **Why powerful:** A high-level *grammar* of interactive graphics — layering, faceting, windowed transforms, aggregations, and cross-filtering expressed declaratively, producing interactive, publication-grade charts far beyond stock chart libraries. It's the engine under Altair, Streamlit charts, and countless dashboards.
- **Why people bounce off:** You author charts as verbose JSON specs and must internalize the grammar — encodings, marks, transforms, selection params, scales. Analysts who can describe the chart in one sentence routinely give up and fall back to an Excel chart because hand-writing and debugging the spec for anything with a moving average, a dual axis, or a linked selection is genuinely painful.
- **AI operator:** The cleanest NL-operator fit in the whole batch — an operator that emits schema-valid Vega-Lite from a plain-language chart request, validates against the published JSON Schema, renders, and then *iterates* ("now log-scale the y-axis", "add a trendline", "color by region") without the user ever touching JSON.
- **Killer example:** user says *"Monthly revenue as a line chart with a 3-month moving average overlay, colored by region"* → AI produces a valid spec with a line mark, a `window` transform for the moving average, and a color encoding, validates it against the schema, and renders — then swaps to log scale on the next sentence.
- **Integration path:** JSON spec validated against the published JSON Schema, plus the JS runtime and the Python `altair` API and Vega Editor for round-tripping.
- **Scores:** demand **5**, buildability **5**.

## 2. Sonarr / Radarr — the config that spawned an entire wiki project

- **Why powerful:** End-to-end media automation — monitors for wanted TV/movies, grades every candidate release against tunable quality and custom-format scoring, grabs from indexers, hands off to a download client, then renames and imports. Paired with Prowlarr it runs an entire library hands-free. The backbone of the modern homelab media stack.
- **Why people bounce off:** Quality Profiles + Custom Formats + release scoring. Getting "1080p remux but not x265, prefer these release groups, avoid hardcoded subs, cap the size" right means hand-building dozens of regex Custom Formats with numeric scores. The entire TRaSH-Guides project exists precisely *because* nobody can configure this correctly by hand — that's the signal.
- **AI operator:** An operator pre-loaded with the *arr v3 REST schema and TRaSH-guide knowledge that translates a plain-English quality preference into the full graph of Custom Formats, scores, Quality Profiles, and release profiles, and applies them consistently across Sonarr and Radarr at once.
- **Killer example:** user says *"I want 1080p Bluray remuxes, skip anything with baked-in subtitles, prefer HEVC under 15 GB, and never grab an HDR-only file my TV can't tone-map"* → AI creates the matching Custom Formats with scoring, assembles a Quality Profile, sets the size limit, and assigns it to every series and movie via the API.
- **Integration path:** REST API v3 (documented at sonarr.tv/radarr.video), API-key auth, plus Prowlarr's API for indexer management.
- **Scores:** demand **5**, buildability **4**.

## 3. OpenRefine — a bare grid and no idea where to start

- **Why powerful:** The dedicated power tool for wrangling messy tabular data — faceting, key-collision clustering, GREL transforms, reconciliation against external databases, and mass edits across millions of cells that would be impossible by hand in a spreadsheet.
- **Why people bounce off:** The entire workflow hinges on learning GREL expressions and the mental model of *facets + clustering algorithms* (fingerprint, ngram, metaphone). Ordinary users open it, see a bare grid, and have no idea how to turn "fix the inconsistent country names" into the right sequence of facet → cluster → transform.
- **AI operator:** An operator pre-loaded with GREL and OpenRefine's operation grammar that translates cleaning intent into the actual apply-operations JSON, runs it through the REST API, and returns a before/after diff of distinct values so the human approves the change rather than authoring it.
- **Killer example:** user says *"Clean up the Country column — merge every spelling of 'United States' and fix the capitalization"* → AI runs a text facet, applies key-collision clustering, issues the mass-edit + text-transform operations, and reports "reduced 47 distinct values to 12" with the diff.
- **Integration path:** HTTP/REST API (`/command/core/*`, apply-operations JSON) plus the `openrefine-client` Python library.
- **Scores:** demand **5**, buildability **4**.

## 4. Inkscape — "make me a logo" vs. node-by-node béziers

- **Why powerful:** A full SVG vector editor — bézier paths, boolean operations, live path effects, bitmap tracing, clipping/masking, and a scriptable object model. Anything from a logo to a laser-cut stencil to a multi-page cut file.
- **Why people bounce off:** Ordinary users want "make me a logo / sticker / stencil" and instead hit node-by-node bézier editing, the path/XML mental model, boolean operations, and the layers/clip/mask system. The gap between intent and vector mechanics is enormous — most people trace once, get frustrated, and leave.
- **AI operator:** An operator that emits and edits real SVG and then drives Inkscape's action pipeline to perform *true* vector operations (path booleans, bitmap trace, object-to-path, align/distribute) instead of raster approximations — so the output is a clean, editable vector, not a flattened image.
- **Killer example:** user says *"Trace this photo into a 3-color vector stencil and center it on an A4 page"* → AI runs the bitmap-trace + posterize actions, boolean-merges the color layers, aligns to page center, and exports print-ready SVG/PDF.
- **Integration path:** Inkscape 1.x CLI `--actions` / interactive shell mode, plus Python extensions via the `inkex` library.
- **Scores:** demand **5**, buildability **4**.

## 5. GDAL / OGR — 40 CLI tools and order-sensitive flag soup

- **Why powerful:** The universal geospatial translator/processor behind nearly all GIS and remote-sensing software — reprojects, warps, mosaics, clips, tiles, and converts 200+ raster/vector formats at scale. If a map operation exists, GDAL does it.
- **Why people bounce off:** It's a bag of ~40 CLI utilities (`gdalwarp`, `gdal_translate`, `ogr2ogr`, `gdalbuildvrt`) with dense, order-sensitive flag soup (`-t_srs EPSG:3857 -cutline … -crop_to_cutline -of COG`) and no GUI. Users can't remember which tool plus which flags do what — the FFmpeg-for-geospatial problem exactly.
- **AI operator:** An NL-to-command layer that picks the right utility, resolves CRS/format/resampling flags, and chains multi-step pipelines (warp → mosaic → cloud-optimized GeoTIFF) into one runnable command sequence, explaining each step.
- **Killer example:** user says *"Reproject these 40 GeoTIFFs to Web Mercator, clip them to this shapefile, and merge into one cloud-optimized GeoTIFF"* → AI chains `gdalwarp -t_srs EPSG:3857 -cutline`, then `gdalbuildvrt` + `gdal_translate -of COG`, and runs it over the batch.
- **Integration path:** CLI utilities (plus the new unified `gdal` command) and Python bindings (`osgeo.gdal`/`ogr`).
- **Scores:** demand **5**, buildability **4**.

## 6. Splunk — the answers are all locked behind SPL

- **Why powerful:** Industry-standard log analytics and SIEM — ingests any machine data and answers almost anything via SPL: search, correlate, `stats`, `transaction`, ML, over billions of events. It's the system of record for security and ops in a huge share of enterprises.
- **Why people bounce off:** SPL is a dense pipe-based query language. Ordinary engineers can't compose `stats` / `eval` / `transaction` / `tstats` correctly, so they reuse a handful of saved searches someone else wrote and never ask the questions they actually have. (Closed and expensive — but that's exactly why unlocking it for non-experts has real budget behind it.)
- **AI operator:** An operator that translates plain-English questions into correct SPL, runs them via the search/REST API, returns the table, and offers to persist the good ones as saved searches, alerts, or dashboard panels.
- **Killer example:** user says *"Show failed SSH logins from outside our IP range in the last 24h, grouped by source IP and user, only IPs with more than 5 attempts"* → AI writes the SPL, runs it, returns the ranked table, and offers to save it as an alert.
- **Integration path:** REST API (search-jobs endpoint), SPL as the artifact, and the official Splunk SDKs.
- **Scores:** demand **5**, buildability **4**.

## 7. Kafka Connect — connector JSON is a rite of passage nobody finishes

- **Why powerful:** Streams data between Kafka and external systems (databases, S3, Elasticsearch, …) via source/sink connectors with exactly-once delivery, schema handling, and single-message transforms — all config, no code.
- **Why people bounce off:** Connectors are configured through sprawling JSON — converters, SMT chains, key/value schemas, topic routing — and failures surface as cryptic serialization/converter mismatches. Getting a Debezium CDC source or an S3 sink working is the canonical "give up halfway" task.
- **AI operator:** An operator pre-loaded with connector configs and the Connect REST API that writes and patches connector JSON, applies the right converters and SMTs, deploys, and then *monitors task status* and explains the converter mismatch when it fails — the failure diagnosis is half the value.
- **Killer example:** user says *"Stream inserts from my MySQL payments table into the payments topic with the schema, using Debezium"* → AI writes the Debezium MySQL source config (server id, table include-list, converters, history topic), POSTs it to the Connect REST API, and reports task health.
- **Integration path:** Kafka Connect REST API (plus `kafka-topics` / admin CLI for the surrounding topics).
- **Scores:** demand **5**, buildability **4**.

## 8. Taskwarrior — enterprise task logic nobody gets past `task add`

- **Why powerful:** A CLI task engine with dependencies, recurrence, user-defined attributes, virtual tags, custom reports, urgency scoring, and on-add/on-modify hooks — genuinely enterprise-grade task logic, in a terminal.
- **Why people bounce off:** The cryptic filter/modification syntax and the `.taskrc` report/UDA configuration mean most people never get past `task add`. Recurrence, dependencies, and custom reports are effectively inaccessible — the power is real but the grammar hides it.
- **AI operator:** An operator that speaks the full command grammar and `.taskrc` schema, translating plain requests into precise commands, custom report definitions, and config edits — and because the CLI is deterministic and scriptable, it's exceptionally cheap to drive correctly.
- **Killer example:** user says *"Every weekday nag me to review the sprint board, tag it work, high priority, and mark it blocked until standup is done"* → AI runs the `task add … recur:weekdays priority:H +work depends:<id>` sequence and confirms the recurrence template.
- **Integration path:** the `task` CLI (deterministic, scriptable) + on-add/on-modify hooks + Taskchampion sync / `task export|import` JSON.
- **Scores:** demand **4**, buildability **5**.

## 9. SuiteCRM — a CRM whose power lives in admin screens sales never open

- **Why powerful:** A full open-source CRM (SugarCRM fork) — accounts, contacts, leads, opportunities, quotes, a Studio module builder, a relationship engine, and a workflow/process-automation engine. Feature-parity with paid CRMs.
- **Why people bounce off:** The dated, dense UI plus Studio (custom modules/fields/relationships) and the Workflow builder are where ordinary sales users bounce off. Bulk-importing messy leads and linking them to the right accounts, or building even a simple automation, means navigating admin screens most SMB owners never learn.
- **AI operator:** An operator that speaks CRM and drives the SuiteCRM v8 REST surface (JSON:API / OAuth2) — creating, deduping, and linking records, running filtered queries, logging activities, and moving deals through the pipeline in plain language.
- **Killer example:** user says *"Import these 200 business cards as leads, create any missing accounts, and link each lead to its company"* → AI parses the list, upserts accounts, creates the leads, sets the account↔lead relationships via REST, and reports which duplicates it merged.
- **Integration path:** REST API v8 (JSON:API, OAuth2), with the legacy v4.1 API as a fallback.
- **Scores:** demand **4**, buildability **4**.

## 10. Ghidra — a stripped binary and an expert-only grind

- **Why powerful:** The NSA's full software reverse-engineering suite — a multi-architecture disassembler and decompiler with data-type recovery, cross-references, and a complete scripting API, used for malware analysis and vulnerability research.
- **Why people bounce off:** Newcomers face a dense multi-panel IDE, must manually navigate assembly and decompiler output, define types, and rename symbols. Making sense of a stripped binary is a steep, slow, expert-only grind with no "just tell me what this does" path.
- **AI operator:** A reverse-engineering co-pilot that runs headless analysis, walks the call graph to find and label functions, and applies renames, comments, and type annotations through the scripting API — turning the blank decompiler view into an annotated one.
- **Killer example:** user says *"Open this malware binary, find the string-decryption routine, and rename it and its variables to something readable"* → AI runs headless analysis, locates the XOR/decrypt function, and applies the renames and comments via the scripting API.
- **Integration path:** the headless analyzer (`analyzeHeadless`), the Java/Python (Jython) scripting API, and PyGhidra / Ghidra-Bridge.
- **Scores:** demand **4**, buildability **4**.

## 11. Magento Open Source — one product variant is a multi-screen ordeal

- **Why powerful:** An enterprise-grade e-commerce engine — configurable/bundle products, multi-store/multi-website hierarchies, catalog and cart price rules, tiered and customer-group pricing, and a full EAV attribute system. It can model almost any storefront.
- **Why people bounce off:** Merchants drown in the EAV data model (attribute sets vs. attribute groups), the store-view/website tree, and the requirement to reindex and flush cache after nearly every config change. Setting up a single configurable product with size/color variants is a multi-screen ordeal that defeats non-technical shop owners.
- **AI operator:** An operator pre-loaded with the Magento catalog/inventory domain that drives the REST API and `bin/magento` CLI — building attribute sets, generating the variation matrix, wiring categories and price rules, and auto-running reindex/cache-flush so the merchant never touches the admin grid.
- **Killer example:** user says *"Add a T-shirt in red and blue, sizes S–XL, $25 each, 10 in stock per variant, and put it in the Summer Sale category"* → AI creates/reuses the color + size attributes and attribute set, generates the configurable parent plus 8 child SKUs via REST, sets stock, assigns the category, then triggers reindex and cache flush.
- **Integration path:** REST API (`/rest/V1/…`) + GraphQL + `bin/magento` CLI.
- **Scores:** demand **5**, buildability **3**.

## 12. authentik — an abstract flow/stage graph that defeats admins

- **Why powerful:** A full self-hosted identity provider — OAuth2/OIDC, SAML, LDAP, SCIM, and forward-auth for reverse proxies, with a flexible flow/stage engine that can model almost any auth journey (enrollment, MFA, recovery, conditional policies).
- **Why people bounce off:** The Flow → Stage → Policy → Binding model. Standing up "Google login with MFA for admins only" means wiring an OAuth source, an authentication flow, TOTP/WebAuthn stages, policy bindings, a provider, an application, and outposts — an abstract, order-dependent object graph that regularly defeats even experienced admins.
- **AI operator:** An operator that knows authentik's flow/stage primitives and its Blueprint (YAML config-as-code) format, composing a working flow + provider + application from a spoken requirement and applying it declaratively via Blueprints or the API.
- **Killer example:** user says *"Let staff log in with our Google Workspace, but force a hardware-key or TOTP prompt before anyone reaches the Grafana app"* → AI creates the OAuth source, an authentication flow with a conditional MFA stage bound to that application, and the OIDC provider + application objects.
- **Integration path:** full REST API (OpenAPI schema) + declarative Blueprints (YAML).
- **Scores:** demand **4**, buildability **3**.
